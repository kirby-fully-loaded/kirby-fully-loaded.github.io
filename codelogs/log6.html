Code Reference

What did you look up?

Shapes are relevant to my major project, so for this Code Log I went to several reference sites and performed a search for "shape." The results were of varying utility.

Which sites did you consult?

I consulted the following sites:

  • W3 Schools How-to page on CSS shapes
  • MDN page on CSS Shapes
  • caniuse results for "shapes"
  • Stack Overflow results for "shapes"

What are the differences in the information presented on each site?

  • W3 Schools presented a list of basic shapes possible through CSS code. The list was rather limited, but this may have been due to the search result I selected.
  • MDN's results for the same search terms presented a much more comprehensive page, which included links to different methods of creatinhg shapes, as well as related topics and additional resources.
  • caniuse did not provide the information I was looking for, so it was not useful for my needs. However, I can see it being useful if you already know how to write a given piece of code, and just want to check its accessibility in terms of browser support.
  • Stack Overflow was not the type of resource I was looking for because I was working on a strict deadline. However, due to its nature I can see it being very useful if one has more time and one has the knowledge to ask effective questions.

What features do you think a code reference page should have?

I think at a minimum a code reference page should have a user-friendly list of topics, a search function, and learner-friendly language to facilitate learning code. Links to related topics are also extremely useful.

Do any of your 4 resources stand out as particularly good or bad?

MDN stood out as particularly useful given the information I was looking for. I'll be studying their results again when I have more time, and may implement some of their information in designing my major project pages.

I'm tempted to say that caniuse stood out as bad, but the reality is it isn't designed to provide the information I was looking for. If I were searching for the information it provided, it would be a very useful, easily-understood site.
